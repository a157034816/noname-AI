# 身临其境的AI

这是一个**仅对本地单机 AI 生效**的无名杀扩展，主要提供：

- 人格/心智模型 + 可视化 AI 面板（头像旁 `AI` 标记）
- 身份局独立身份猜测（不读取引擎真实身份）
- 反全知（盲选手牌随机化）、评分噪声（仅冲动型）
- 身临其境：视觉调试（出牌/弃牌阶段按 AI 打分高亮推荐选择，默认关闭）
- `scripts/` 脚本插件系统（可在“脚本插件管理”里启用/禁用/排序；保存后通常需重启）
- 检查更新/更新（可跟踪 stable 正式版或从可选列表里选择指定 PR 测试版；在支持写文件的环境可一键下载覆盖更新，更新后需重启生效）

说明：

- 游戏内：扩展设置 → 帮助

## 更新通道

扩展设置里可配置两种更新通道：

- `stable 正式版`：跟踪 GitHub Releases 的正式发布
- `PR 测试版`：通过可选 PR 列表选择一个开放且已有测试包的 PR，只跟踪该 PR 的测试发布

注意事项：

- stable 和 PR 之间切换时，会强制重新下载安装当前扩展
- 从一个 PR 切换到另一个 PR，也会强制重新下载安装当前扩展
- PR 测试版版本号独立于 stable，从 `0.0.1-pr.<pr>.<run>` 起算
- 仓库会为**本仓库 PR**自动创建测试版 prerelease；fork PR 不自动发测试版

## 身临其境：视觉调试（Debug）

开关位置：`扩展设置 → 身临其境的AI → 身临其境：视觉调试(Debug)`（默认关闭；支持热切换，推荐重启更稳）。

启用后效果（仅对你当前操控且 `event.isMine()` 的交互选择 UI 生效）：

- 出牌阶段：高亮“如果现在由 AI 操作，最可能出的牌”，并用指示线指向预测目标
- 弃牌阶段：用不同边框高亮区分“建议弃置/建议留下”

手动验收步骤：

- 开启开关并重启游戏
- 进入对局：在自己的出牌阶段（未托管也会显示）观察手牌区是否出现 `AI` 高亮与指示线
- 回合末进入弃牌阶段时，观察手牌边框是否出现两种高亮（弃置/留下）

已知限制：

- 多目标/分步选目标的牌：目前仅显示“单目标近似”的指示线
- 无目标牌：仅高亮卡牌本体，不显示指示线
